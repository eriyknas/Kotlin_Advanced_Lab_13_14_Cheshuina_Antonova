## Generics
### Generics позволяют:
1. описать класс один раз
2. подставлять нужный тип данных при создании объекта
3. избежать дублирования кода
4. повысить читаемость и безопасность типов
### Пример
```bash
class Question<T>(
    val questionText: String,
    val answer: T,
    val difficulty: Difficulty
)
```
## Enum классы
### Enum class применяется, когда:
1. существует ограниченный набор возможных значений
2. использование любых других значений недопустимо
3. требуется строгий контроль на уровне компилятора
### Пример
```bash
enum class Difficulty {
    EASY, MEDIUM, HARD
}
```
###  Преимущества использования enum class:
1. защитаотопечаток
2. контроль допустимых значений на этапе компиляции
3. упрощение поддержки и рефакторинга
4. болеечитаемый и самодокументируемый код
5. активное использование в Android-разработке (состояния, события, режимы UI)
## Data классы
Data class — это класс, предназначенный для хранения данных.
### Если класс объявлен как data class, компилятор Kotlin автоматически генерирует для него следующие методы:
1. equals()
2. hashCode() (важен при работе с коллекциями)
3. toString()
4. componentN() (для деструктуризации)
5. copy()
### Пример
```bash
data class Question<T>(
    val questionText: String,
    val answer: T,
    val difficulty: Difficulty
)
```
## Singleton 
Singleton — это класс, у которого может существовать только один экземпляр.
### Пример
```bash
object StudentProgress {
    var answered = 0
    var total = 10
}
```
## Companion Object
ВKotlin объекты можно объявлять внутри классов.
Если такой объект помечен как companion, его свойства доступны напрямую
через имя класса.
### Пример
```bash
class Quiz {
    companion object {
        val answered = 5
        val total = 10
    }
}
```
##  Extension-свойства и функции
### Extensions позволяют:
1. добавлять новые свойства и функции к уже существующим типам
2. использовать их через точечную нотацию (.), как будто они были частью класса
3. неизменять исходный код класса
### Важно:
1. расширения не изменяют сам класс
2. онилишьдобавляют удобный синтаксис поверх него
### Пример
```bash
val Quiz.StudentProgress.progressText: String
    get() = "${Quiz.answered} of ${Quiz.total} answered"
```
### Интерфейсы
### Интерфейсы широко используются в Kotlin и Android-разработке, например для:
1. внедрения зависимостей (Dependency Injection);
2. подменыреализаций в тестах (mock-объекты);
3. работысразными платформами (Compose Multiplatform);
4. расширения возможностей без изменения исходного кода (например,
Modifier в Compose).
### Пример
```bash
interface ProgressPrintable {
    val progressText: String
    fun printProgressBar()
}
```
## Scope-функции
Scope-функции позволяют работать со свойствами и методами объекта, не
повторяя имя переменной, если это избыточно. Это особенно часто используется в
Android-разработке и Jetpack Compose.
### Scope-функции — это функции высшего порядка, которые:
1. принимают лямбда-выражение
2. временно меняют контекст (scope) внутри этого блока
3. позволяют обращаться к объекту через it или this
### Пример
```bash
question1.let {
    println(it.questionText)
    println(it.answer)
}
```
## Коллекции
### Резюмируем:
1. **Arrays** хранят упорядоченные данные того же типа и имеют фиксированный
размер.
2. **Lists**- это изменяемая, упорядоченная коллекция.
3. **Sets** являются неупорядоченными коллекциями и не могут содержать
дубликатов.
4. **Maps** работают аналогично наборам и хранящимся парам ключей и значений
указанного типа.
### Примеры
```bash
val planets = listOf("Mercury", "Venus", "Earth")
val set = mutableSetOf("Pluto", "Mars")
val map = mutableMapOf("Mercury" to 0, "Venus" to 0)
```
## Функции высокого порядка
Функции высокого порядка (high order function)- это функции, которые либо
принимают функцию в качестве параметра, либо возвращают функцию, либо и то, и
другое.
### Примеры
```bash
cookies.forEach { println(it.name) }
val prices = cookies.map { it.price }
val softCookies = cookies.filter { it.softBaked }
```
